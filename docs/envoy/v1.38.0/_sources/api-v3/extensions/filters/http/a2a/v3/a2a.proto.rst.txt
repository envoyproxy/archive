===========

.. _extension_envoy.filters.http.a2a:

This extension has the qualified name ``envoy.filters.http.a2a``


.. note::
  This extension is functional but has not had substantial production burn time,
  use only with this caveat.

  This extension has an unknown security posture and should only be
  used in deployments where both the downstream and upstream are

.. _envoy_v3_api_field_extensions.filters.http.a2a.v3.A2a.max_request_body_size:


max_request_body_size
  (`UInt32Value <https://developers.google.com/protocol-buffers/docs/reference/google.protobuf#uint32value>`_) Maximum size of the request body to buffer for JSON-RPC validation.
  If the request body exceeds this size, the request is rejected with ``413
  Payload Too Large``. This limit applies to both ``REJECT`` and
  ``PASS_THROUGH`` modes to prevent unbounded buffering.

  It defaults to 8KB (8192 bytes) and the maximum allowed value is 10MB
  (10485760 bytes).

  Setting it to 0 would disable the limit. It is not recommended to do so in
  production.

.. _envoy_v3_api_field_extensions.filters.http.a2a.v3.MethodParsingConfig.group:


group
  (`string <https://developers.google.com/protocol-buffers/docs/proto#scalar>`_) The group/category name to assign to this method (e.g., "tasks", "message").
  If provided, this overrides any built-in classification for the method.
  This will be emitted to dynamic metadata under the key specified by ``group_metadata_key``.


.. _envoy_v3_api_field_extensions.filters.http.a2a.v3.MethodParsingConfig.paths:


paths
  (**repeated** `string <https://developers.google.com/protocol-buffers/docs/proto#scalar>`_) List of attributes to extract for this method, specified by their JSON paths (e.g., "params.name").

.. _envoy_v3_api_field_extensions.filters.http.a2a.v3.ParserConfig.method_configs:


method_configs
  (**repeated** map<`string <https://developers.google.com/protocol-buffers/docs/proto#scalar>`_, :ref:`extensions.filters.http.a2a.v3.MethodParsingConfig <envoy_v3_api_msg_extensions.filters.http.a2a.v3.MethodParsingConfig>`>) Method-specific overrides for grouping and attribute extraction.
  If a method is not specified in method_configs, or if 'group' is empty in its config,
  its group will be determined by built-in classification based on method prefix
  (e.g., "message" for "message/send") and default extraction rules will be applied for that method.


.. _envoy_v3_api_field_extensions.filters.http.a2a.v3.ParserConfig.always_extract_attributes:


always_extract_attributes
  (**repeated** `string <https://developers.google.com/protocol-buffers/docs/proto#scalar>`_) Attributes that should always be extracted regardless of the method.
  Specified by their JSON paths (e.g., "params.id").


.. _envoy_v3_api_field_extensions.filters.http.a2a.v3.ParserConfig.group_metadata_key:


group_metadata_key
  (`string <https://developers.google.com/protocol-buffers/docs/proto#scalar>`_) The dynamic metadata key under which the method's group name will be stored
  (e.g., "a2a_group"). If this key is empty, group information will not be
  added to dynamic metadata.
